# Project Narrative

Kay-Ledger is a finance-grade backend portfolio project.

It starts like a marketplace backend:

- workspaces
- actors
- sellers
- buyers
- catalog
- offerings
- bookings
- payments

Then it moves into the harder parts:

- double-entry ledger
- settlement
- payouts
- refunds
- disputes
- subscriptions
- async event delivery
- provider callback chaos
- reconciliation
- operator recovery
- financial close
- maker-checker approvals
- evidence packs
- multi-region resilience

## The core theme

The core theme is **financial truth**.

In a financial backend, the system must answer:

- What happened?
- Who caused it?
- Which account was affected?
- Which ledger entries prove it?
- Which provider events support or contradict it?
- Was the event duplicated?
- Was it replayed?
- Was it repaired?
- Was it approved?
- Was it after close?
- Which evidence pack explains it?
