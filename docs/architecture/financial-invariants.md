# Financial Invariants

Financial systems are built around invariants.

## Core invariants

- Ledger entries must balance.
- Money movement must be immutable.
- Balances are derived from ledger truth.
- Duplicate requests must not double-move money.
- Provider callbacks must be deduped.
- Provider truth must be reconciled.
- Closed periods must block ordinary mutation.
- Frozen funds must not be paid out.
- Sensitive actions need approval.
- Wrong-region writes must be fenced.
