# Test Naming Guide

Good tests describe protected financial behavior.

## Examples

- `duplicateCaptureDoesNotDoublePostLedgerEntries`
- `providerCallbackReplayIsDedupedAndAudited`
- `closedAccountingWindowRejectsOrdinaryMutation`
- `wrongRegionCannotMutateWorkspaceOwnedByPeer`
- `makerCannotApproveOwnSensitiveFinancialAction`
